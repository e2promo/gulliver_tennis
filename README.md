# Первенство по настольному теннису — Парк Гулливер

Одностраничный сайт для регистрации участников городского турнира по настольному теннису в парке культуры и отдыха «Гулливер» (г. Донецк, ДНР).

Сайт включает:
- страницу объявления с информацией о турнире,
- устав и регламент соревнований,
- форму регистрации участников,
- панель администратора для управления заявками.

## Технологии

- HTML + CSS + JavaScript (vanilla)
- Vercel KV (Redis) для хранения заявок участников
- Vercel Serverless Functions (`/api/registrations.js`)

## Деплой на Vercel

### 1. Загрузить на GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/gulliver-tennis.git
git push -u origin main
```

### 2. Задеплоить на Vercel

1. Перейдите на [vercel.com](https://vercel.com) и войдите через GitHub.
2. Нажмите **Add New → Project**.
3. Выберите репозиторий `gulliver-tennis`.
4. Нажмите **Deploy**.

После деплоя вы получите адрес вида `https://gulliver-tennis-xxx.vercel.app`.

### 3. Подключить базу данных Vercel KV

1. В [dashboard.vercel.com](https://vercel.com/dashboard) откройте проект → раздел **Storage**.
2. Нажмите **Create Database** → выберите **KV**.
3. Назовите базу, например `gulliver-kv`, выберите регион `Frankfurt` и нажмите **Create**.
4. На странице созданной базы перейдите во вкладку **Projects** и нажмите **Connect Project** → выберите `gulliver-tennis`.
   - Переменные окружения (`KV_REST_API_URL`, `KV_REST_API_TOKEN` и др.) добавятся автоматически.

### 4. Добавить переменную ADMIN_TOKEN

1. В Vercel Dashboard → ваш проект → **Settings → Environment Variables**.
2. Добавьте переменную:
   - **Name:** `ADMIN_TOKEN`
   - **Value:** `ghDTfdSTkzESSpBPcJy9`
3. Нажмите **Save**.

### 5. Передеплоить проект

После добавления переменных нужен повторный деплой:

- Vercel Dashboard → ваш проект → вкладка **Deployments**.
- Нажмите на последний деплой → **Redeploy**.
- Либо сделайте любой коммит в GitHub — деплой произойдёт автоматически.

## Готово ✅

Теперь сайт доступен из любого браузера и с любого устройства. Все заявки сохраняются в единой базе данных Vercel KV.

### Проверка работы

1. Откройте сайт → перейдите на **«Регистрация участников»**.
2. Заполните форму и отправьте заявку.
3. Перейдите на **«Админ»** и войдите с паролем: `ghDTfdSTkzESSpBPcJy9`.
4. Заявка должна отобразиться в таблице.
5. Откройте сайт с другого устройства — данные будут синхронизированы.

## Структура проекта

| Файл / Папка | Назначение |
|---|---|
| `index.html` | Главный файл сайта (объявление, устав, форма регистрации, админка) |
| `api/registrations.js` | Serverless API для сохранения, чтения и удаления регистраций |
| `vercel.json` | Конфигурация Vercel |
| `package.json` | Зависимости (пакет `@vercel/kv`) |
| `.gitignore` | Исключения для Git (секреты, локальные env-файлы) |
| `SETUP.md` | Подробная инструкция по настройке Vercel KV |

## Безопасность

- Пароль администратора хранится в переменной окружения `ADMIN_TOKEN` на Vercel и в коде фронтенда.
- При удалении заявок API проверяет токен в заголовке `X-Admin-Token`.
- Секретные файлы (`.env*.local`) исключены из репозитория.
