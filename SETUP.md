# 🎾 Настройка хранилища данных на Vercel KV

Это руководство объясняет, как подключить базу данных Vercel KV к вашему проекту,  
чтобы регистрации сохранялись **постоянно** и были видны **с любого устройства**.

---

## Шаг 1 — Загрузить проект на GitHub

Если проект ещё не на GitHub:

1. Зайдите на [github.com](https://github.com) → **New repository**
2. Назовите репозиторий, например `gulliver-tennis`
3. В папке проекта выполните в терминале:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/ВАШ_ЛОГИН/gulliver-tennis.git
   git push -u origin main
   ```

---

## Шаг 2 — Задеплоить проект на Vercel

1. Зайдите на [vercel.com](https://vercel.com) и войдите через GitHub
2. Нажмите **Add New → Project**
3. Выберите ваш репозиторий `gulliver-tennis`
4. Нажмите **Deploy** (настройки по умолчанию подойдут)
5. Дождитесь деплоя — вы получите URL вида `https://gulliver-tennis-xxx.vercel.app`

---

## Шаг 3 — Создать базу данных Vercel KV

1. В [dashboard.vercel.com](https://vercel.com/dashboard) перейдите в раздел **Storage**
2. Нажмите **Create Database** → выберите **KV**
3. Назовите базу, например `gulliver-kv`
4. Регион: выберите ближайший (например `Frankfurt` для Европы)
5. Нажмите **Create**

---

## Шаг 4 — Привязать KV к проекту

1. На странице созданной KV базы перейдите во вкладку **Projects**
2. Нажмите **Connect Project** → выберите `gulliver-tennis`
3. Подтвердите — переменные окружения добавятся автоматически:
   - `KV_URL`
   - `KV_REST_API_URL`
   - `KV_REST_API_TOKEN`
   - `KV_REST_API_READ_ONLY_TOKEN`

---

## Шаг 5 — Добавить переменную ADMIN_TOKEN

Это секретный токен для защиты операций удаления заявок:

1. В Vercel → ваш проект → **Settings → Environment Variables**
2. Добавьте переменную:
   - **Name:** `ADMIN_TOKEN`
   - **Value:** `ghDTfdSTkzESSpBPcJy9` (или любой другой пароль)
3. Нажмите **Save**

> ⚠️ Это значение должно совпадать с `ADMIN_TOKEN` в `index.html` (строка ~1776)

---

## Шаг 6 — Передеплоить проект

После добавления переменных нужен повторный деплой:

1. Vercel → ваш проект → вкладка **Deployments**
2. Нажмите на последний деплой → **Redeploy**
3. Или сделайте любой коммит в GitHub — Vercel задеплоит автоматически

---

## ✅ Проверка

После деплоя:

1. Откройте сайт → перейдите на страницу **Регистрация**
2. Заполните и отправьте форму
3. Войдите в **Админку** (пароль: `ghDTfdSTkzESSpBPcJy9`)
4. Откройте сайт с **другого устройства или браузера**
5. Войдите в админку — заявка должна отображаться ✓

---

## Возникли проблемы?

- **"KV_URL is not defined"** → переменные не добавились, повторите Шаг 4
- **"401 Unauthorized" при удалении** → проверьте что `ADMIN_TOKEN` в env совпадает с кодом
- **Данные не появляются** → проверьте DevTools → Network → запрос на `/api/registrations`
